# AGENTS.md

## Repository Shape
- This is a documentation-only repo for Pactus, not the main application codebase.
- The root `docs/` tree contains the OpenAPI contract; `docs/openapi.yaml` is the entrypoint and references module files under `docs/modules/` plus shared errors in `docs/schemas/errors.yaml`.
- The documentation site is the Astro/Starlight project in `astro/`; there is no root `package.json`.
- Starlight content lives under `astro/src/content/docs/es/`, with Spanish as the configured default and only locale.

## Commands
- Run npm commands from `astro/`, not the repo root.
- Install dependencies with `npm install` in `astro/`; the checked-in lockfile is `astro/package-lock.json`.
- Start local docs: `npm run dev` from `astro/`.
- Verify the site build: `npm run build` from `astro/`.
- Preview a production build: `npm run preview` from `astro/`.
- Bundle OpenAPI: `npx @redocly/cli bundle ./docs/openapi.yaml --output openapi.bundle.yaml` from the repo root.
- There are no configured lint, formatter, typecheck, or test scripts; use `npm run build` as the main executable verification.

## Editing Notes
- Update OpenAPI paths in `docs/openapi.yaml`; keep endpoint details split into the existing module files under `docs/modules/<domain>/`.
- Reuse shared error schemas from `docs/schemas/errors.yaml` instead of duplicating error response shapes in module files.
- Starlight sidebars are autogenerated by directory from `astro/astro.config.mjs`; new docs pages should go in one of the configured Spanish directories (`producto`, `arquitectura`, `frontend`, `backend`, `base-datos`, `ia`) or the config must be updated.
- Keep documentation language Spanish unless editing an existing English fragment for consistency.

## Commit Convention
- Existing repo docs specify Conventional Commit prefixes: `fix:`, `feat:`, `feat!:`, `style:`, `docs:`, `refactor:`, `chore:`, `ci:`, and `test:`.
