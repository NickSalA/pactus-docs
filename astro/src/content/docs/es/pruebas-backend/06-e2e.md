Las pruebas end-to-end (E2E) verifican el funcionamiento completo de la aplicación desde la perspectiva del usuario. Estas pruebas inicializan la aplicación real y prueban múltiples componentes juntos.

## Archivos de Pruebas E2E

Las pruebas E2E se encuentran en la raíz del directorio `tests/`:

| Archivo | Descripción |
|--------|-------------|
| test_app_setup.py | Setup y configuración de la aplicación |
| test_chatbot_prompt.py | Validación de prompts del agente |

## test_app_setup.py

Archivo: `tests/test_app_setup.py`

Este archivo contiene pruebas que verifican el setup correcto de la aplicación FastAPI.

### Pruebas de startup

| Test | Descripción |
|------|-------------|
| `test_app_creates_successfully` | La app se crea sin excepciones |
| `test_root_endpoint` | Endpoint raíz responde 200 |
| `test_documents_router_is_registered` | Router de documentos registrado |
| `test_openapi_schema_is_available` | Schema OpenAPI disponible |

```python
@pytest.fixture
def app() -> FastAPI:
    """Crea la app FastAPI sin lifespan."""
    from contractai_backend.factory import create
    return create()


@pytest.mark.asyncio
async def test_app_creates_successfully(app: FastAPI):
    """La factory debe crear la app sin lanzar excepciones."""
    assert app is not None
    assert isinstance(app, FastAPI)
```

```python
@pytest.mark.asyncio
async def test_root_endpoint(app: FastAPI):
    """El endpoint raíz debe responder 200 con mensaje de bienvenida."""
    async with AsyncClient(transport=ASGITransport(app=app), base_url="http://test") as client:
        response = await client.get("/")
    assert response.status_code == status.HTTP_200_OK
    body = response.json()
    assert "message" in body
    assert "version" in body
```

```python
@pytest.mark.asyncio
async def test_documents_router_is_registered(app: FastAPI):
    """El router de documents debe estar registrado en la app."""
    routes = [route.path for route in app.routes]
    assert any("/documents" in path for path in routes)
```

```python
@pytest.mark.asyncio
async def test_openapi_schema_is_available(app: FastAPI):
    """El schema OpenAPI debe estar disponible."""
    async with AsyncClient(transport=ASGITransport(app=app), base_url="http://test") as client:
        response = await client.get("/openapi.json")
    assert response.status_code == status.HTTP_200_OK
    schema = response.json()
    assert "paths" in schema
    assert "openapi" in schema
```

## test_chatbot_prompt.py

Archivo: `tests/test_chatbot_prompt.py`

Este archivo contiene pruebas que validan los prompts del agente conversacional.

### Pruebas de prompts

| Test | Descripción |
|------|-------------|
| `test_context_prompt_defines_a1_json_routes` | Prompt de contexto define A1 |
| `test_permission_prompt_defines_a2_json_routes` | Prompt de permisos define A2 |
| `test_conversation_prompt_includes_tool_routing_rules` | Prompt de conversación incluye reglas |

```python
def test_context_prompt_defines_a1_json_routes() -> None:
    prompt = get_context_agent_prompt()

    assert "You are A1" in prompt
    assert "n1_early_response" in prompt
    assert "a2_permissions" in prompt
    assert "Return ONLY JSON" in prompt
```

```python
def test_permission_prompt_defines_a2_json_routes() -> None:
    prompt = get_permission_agent_prompt()

    assert "You are A2" in prompt
    assert "trusted backend user context" in prompt
    assert "party_lookup_tool" in prompt
    assert "ADMIN, HR, MANAGER, WORKER" in prompt
    assert "a3_conversation" in prompt
    assert "n2_denied_response" in prompt
    assert "HR can access only LABOR" in prompt
```

```python
def test_conversation_prompt_includes_tool_routing_rules() -> None:
    prompt = get_conversation_agent_prompt()

    assert "currently_active=true" in prompt
    assert "start_date <= today <= end_date" in prompt
    assert "contracts with a specific service" in prompt
    assert "service_name or service_id" in prompt
    assert "which services are attached to a contract" in prompt
    assert "document_ids" in prompt
    assert "### Ranking de clientes" in prompt
```

## Diferencia con otras pruebas

Las pruebas E2E se diferencian de las demás en que:

| Característica | Unitarias | Integración | API | E2E |
|---------------|----------|------------|-----|-----|
| Dependencias externas | No | Mocks | Mocks | Reales/Simuladas |
| Inicialización app | No | No | No | Sí |
| Múltiples capas | No | Una | Una | Todas |
| Tiempo de ejecución | Ms | Ms | Ms | Segundos |
